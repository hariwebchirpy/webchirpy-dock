# Summary

This commit refactors the documentation generation workflow to use OpenRouter for AI-powered documentation creation and updates the project's dependencies.

# Technical Changes

The core change involves updating the CI/CD pipeline (`.github/workflows/auto-docs.yml`) to use `OPENROUTER_API_KEY` instead of `OPENAI_API_KEY`. The `generateCommitDocs.js` script has been modified to import and utilize the `@openrouter/sdk`, along with logic to attempt multiple AI models if the primary one fails. The script now also saves generated documentation directly into the `content/projects/${PROJECT_NAME}/changes/` directory, removing the need for a `temp_docs` intermediate.

Dependency updates are reflected in `package.json` and `package-lock.json`, including additions like `@openrouter/sdk`, new Radix UI components, `cmdk`, `framer-motion`, and `tailwindcss-animate`. The `openai` package is also updated.

# Impact

Developers can now leverage OpenRouter with different AI models for generating documentation. The CI/CD workflow is more robust with the retry logic for AI model selection. The project structure for storing generated documentation has been streamlined. This change introduces new dependencies that might affect build times or bundle sizes.
